## Config Directory

The `.sandcastle/` directory in your repository contains all Sandcastle configuration:

- `config.json` -- Main configuration file
- `Dockerfile` -- Defines the sandbox environment
- `prompt.md` -- The prompt sent to the agent each iteration
- `.env` / `.env.example` -- Environment variables for the sandbox

## Config File

The `config.json` file controls how Sandcastle operates:

```json
{
  "agent": "claude-code",
  "maxIterations": 10
}
```

### Options

| Option          | Description                | Default         |
| --------------- | -------------------------- | --------------- |
| `agent`         | The agent provider to use  | `"claude-code"` |
| `maxIterations` | Maximum iterations per run | `10`            |

## Prompt

The `prompt.md` file is the instruction text passed to the agent at the start of each iteration. It supports:

- **Prompt arguments** -- `{{KEY}}` placeholders replaced with values from `promptArgs`
- **Shell expressions** -- `` !`command` `` markers that evaluate shell commands inside the sandbox

## Environment Variables

Sandcastle resolves environment variables from multiple sources in order:

1. Repository root `.env`
2. Config directory `.env`
3. `process.env`

Only keys declared in a `.env` file are resolved from `process.env`.
